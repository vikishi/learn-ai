Lesson 1-1: The Claude Code Ecosystem

20 min · Concept

Claude Code has four extensibility layers that transform it from a simple chat into a fully configurable development platform:

  1. CLAUDE.md — Project Memory: A markdown file at your project root that persists across sessions and defines conventions, tech stack, and architecture decisions.
  2. Skills — Reusable Prompt Templates: Located at .claude/skills/<name>/SKILL.md, they auto-detect and trigger based on context, and inject live data with !`command` syntax.
  3. Subagents — Parallel AI Workers: Defined in .claude/agents/ files, each with its own context window, system prompt, and model selection.
  4. Hooks — Deterministic Lifecycle Events: Fire at specific points in the workflow, execute real code, and cannot hallucinate because they run deterministic scripts.
bash 
npm install -g @anthropic-ai/claude-code
cd your-project
claude
# Inside Claude Code:
/init    # Generates CLAUDE.md by scanning your repo
/context # See how your context window is being used
/help    # Browse all available commands
💡
Platform, Not Just a Chat
Claude Code is a platform you configure, not just a chat you prompt. The four extensibility layers let you encode your team's entire workflow into repeatable, version-controlled automation.

Lesson 1-2: Writing an Effective CLAUDE.md

25 min · Hands-on

What Belongs in CLAUDE.md

  • Tech stack and versions
  • Architecture overview (directory structure)
  • Coding conventions and patterns
  • Common commands (test, lint, build, deploy)

What Does NOT Belong

  • Long documentation or API references
  • Temporary instructions or one-off tasks
  • Everything — be selective and concise
markdown 
# Project: Acme API

## Tech Stack
- Node.js 20 + TypeScript 5.4
- Express with Zod validation
- PostgreSQL 16 + Drizzle ORM
- Jest for testing, Prettier + ESLint for formatting

## Architecture
- src/routes/ — Express route handlers
- src/services/ — Business logic (no DB access)
- src/repositories/ — Database access layer
- src/middleware/ — Auth, rate limiting, error handling

## Conventions
- Every route handler must validate input with Zod schemas
- Services return Result<T, Error> types, never throw
- All DB queries go through repositories, never raw SQL in services
- Tests live next to source files: user.service.ts → user.service.test.ts

## Commands
- npm test — run all tests
- npm run lint — run ESLint + Prettier check
- npm run migrate — run Drizzle migrations
⚠️
Keep It Short
Keep CLAUDE.md under 200 lines. Every token counts against your context window. A bloated CLAUDE.md means less room for actual code and conversation.

Lesson 1-3: Anthropic Skilljar Courses

15 min · Concept

Anthropic offers a structured Developer Track on Skilljar. Complete these courses in order to build a strong foundation:

Developer Track (Recommended Order)

  1. Claude Code 101 (~1 hour) — Core concepts and basic usage
  2. Claude Code in Action (~3-4 hours) — Real-world workflows and patterns
  3. Introduction to Subagents (~1 hour) — Parallel AI workers
  4. Introduction to Agent Skills (~2 hours) — Reusable automation
  5. Introduction to MCP (~2 hours) — Model Context Protocol
  6. Building with the Claude API (~8 hours) — Full API development

Supplementary Courses

  • Introduction to Claude Cowork
  • AI Capabilities and Limitations

The Automated Pipeline

Chain skills, hooks, and CI/CD into a fully automated development pipeline — from feature request to merged PR with zero manual steps.

3 lessons · ~90 minutes · Advanced

Lesson 5-1: The Full-Circle Workflow

35 min · Hands-on

The ultimate Claude Code workflow chains every skill into a single pipeline that takes a feature from idea to shipped PR:

The Pipeline

  1. Write — Implement the feature code
  2. Self-Review — AI reviews its own code for issues
  3. Generate Tests — Create comprehensive test coverage
  4. Run Tests — Execute tests and fix failures
  5. Lint/Format — Auto-fix style issues
  6. Security Check — Scan for vulnerabilities
  7. Commit — Create a conventional commit
  8. Push & Create PR — Open a pull request with description

The /ship Skill

This skill encapsulates the entire pipeline into a single command:

markdown 
# /ship — Full Pipeline Skill

## Steps

1. **Self-Review** your implementation:
   - Check for bugs, edge cases, error handling
   - Verify naming conventions and code style
   - Fix any issues found

2. **Generate Tests**:
   - Create unit tests for all new functions
   - Add integration tests for API endpoints
   - Aim for >80% coverage on new code

3. **Run Tests**:
   - Execute: `npm test`
   - If failures: fix and re-run (max 3 attempts)
   - All tests must pass before continuing

4. **Lint & Format**:
   - Run: `npm run lint:fix`
   - Run: `npm run format`

5. **Security Scan**:
   - Run: `npm audit --production`
   - Check for hardcoded secrets with grep
   - Block if critical vulnerabilities found

6. **Commit**:
   - Stage all changes: `git add -A`
   - Commit with conventional format: `type(scope): description`
   - Include Co-authored-by trailer

7. **Push & PR**:
   - Push branch: `git push -u origin HEAD`
   - Create PR: `gh pr create --fill`
   - Add labels based on change type

8. **Report**:
   - Summarize what was done
   - List files changed
   - Link to the PR

Headless Mode Execution

Run the entire pipeline non-interactively with headless mode:

bash 
claude -p "implement the user avatar upload feature and /ship" \
  --dangerously-skip-permissions
⚠️
Use --dangerously-skip-permissions Carefully
Only use --dangerously-skip-permissions for well-tested workflows where you trust all hooks and skills. This flag bypasses all permission prompts — Claude will execute any tool without asking. Always pair it with PreToolUse hooks that block dangerous operations.

Lesson 5-2: CI/CD Integration with Claude Code

Reference · Teams/Enterprise only
⚠️
Requires Teams or Enterprise Plan
Running Claude Code headlessly in CI/CD pipelines (claude -p in GitHub Actions, automated PR bots, scheduled pipeline execution) requires a Teams or Enterprise subscription — it is not available on the Pro plan. This section is for awareness only.

What's Possible (Teams/Enterprise)

With a Teams or Enterprise plan, you can run Claude Code non-interactively inside CI/CD:

  • AI code review on every PR — automatically post a review comment when a pull request is opened
  • Auto-generate missing tests — detect untested code and write tests as part of the CI run
  • Security gate in CI — run the /security-gate skill on every merge and block on critical findings
  • Scheduled quality reports — nightly cron jobs for coverage, TODO audits, and dependency scans
  • Pre-commit hooks — lightweight local checks via claude -p before commits land

These workflows use the same skills and subagents you build in this course — just triggered by CI events rather than your terminal.

💡
When You're Ready
If you upgrade to Teams, see the official Claude Code docs for headless mode reference and GitHub Actions examples. The patterns are identical to the /ship skill — just triggered by webhook instead of your terminal.

Lesson 5-3: MCP: Connecting Claude to Your Tools

25 min · Hands-on

The Model Context Protocol (MCP) connects Claude to external tools — issue trackers, databases, monitoring systems — turning it into a fully integrated development assistant.

MCP Server Setup

bash 
# Connect to GitHub (issues, PRs, repos) — stdio server
claude mcp add github -- npx -y @modelcontextprotocol/server-github

# Connect to PostgreSQL (query databases directly) — stdio server
claude mcp add postgres -- npx -y @modelcontextprotocol/server-postgres "postgresql://localhost/mydb"

# Connect to Linear (project management) — SSE server
claude mcp add --transport sse linear https://mcp.linear.app/sse

# Connect to Sentry (error monitoring) — HTTP server
claude mcp add --transport http sentry https://mcp.sentry.dev/mcp

The "Implement from Ticket" Skill

With MCP connections, you can build a skill that goes from ticket to shipped code:

markdown 
# /from-ticket — Implement from Issue Tracker

## Input
- Ticket ID (e.g., LIN-1234 or GH#42)

## Steps

1. **Read Ticket**:
   - Fetch ticket details from Linear/GitHub via MCP
   - Extract requirements, acceptance criteria, and context

2. **Explore Codebase**:
   - Find relevant files based on ticket description
   - Understand existing patterns and architecture

3. **Implement**:
   - Write the code following project conventions
   - Handle edge cases mentioned in ticket

4. **Test**:
   - Generate tests matching acceptance criteria
   - Run full test suite: `npm test`
   - Fix any failures

5. **Ship** (invoke /ship):
   - Self-review, lint, security check
   - Commit, push, create PR
   - Link PR to ticket

6. **Update Ticket**:
   - Move ticket to "In Review" status
   - Add comment with PR link and summary of changes
   - Tag relevant reviewers
💡
The 10x Workflow
This is the 10x workflow: ticket → code → tested → reviewed → shipped → ticket updated. A single command transforms a ticket into a complete, tested, reviewed pull request with the ticket automatically updated. What used to take hours now takes minutes.

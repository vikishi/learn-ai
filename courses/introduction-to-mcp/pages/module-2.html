Module 2

Connecting to Existing Servers

Add public and private MCP servers to Claude Code and share the configuration with your team so everyone gets the same integrations automatically.

2 lessons · ~40 minutes · Hands-on

Lesson 2-1: Adding MCP Servers to Claude Code

20 min · Hands-on

Claude Code module 5 introduced claude mcp add briefly. Here we go deeper — different transport types, verifying connections, and understanding what each server actually exposes.

Two Transport Types

The transport you use depends on how the server is deployed:

  • stdio — Claude spawns the server as a local subprocess. Used with -- npx ... or -- node server.js. No --transport flag needed; the -- separator implies stdio.
  • HTTP/SSE — The server runs remotely. Specify --transport http or --transport sse with the URL.

Adding stdio Servers

bash 
# GitHub — read issues, PRs, repos, search code
claude mcp add github -- npx -y @modelcontextprotocol/server-github

# PostgreSQL — query tables, inspect schemas
claude mcp add postgres -- npx -y @modelcontextprotocol/server-postgres "postgresql://localhost/mydb"

# Filesystem — read/write files outside the project dir
claude mcp add files -- npx -y @modelcontextprotocol/server-filesystem /path/to/dir

Environment variables for secrets go with -e:

bash 
# GitHub with auth token (required for private repos)
claude mcp add github -e GITHUB_TOKEN=your_token -- npx -y @modelcontextprotocol/server-github

Adding HTTP and SSE Servers

bash 
# Sentry — query errors, issues, releases (HTTP)
claude mcp add --transport http sentry https://mcp.sentry.dev/mcp

# Linear — read and write issues, projects, cycles (SSE)
claude mcp add --transport sse linear https://mcp.linear.app/sse

Verifying Your Connections

bash 
# List all configured servers
claude mcp list

# Inspect a specific server (shows its tools and resources)
claude mcp get github

Once connected, you can use the tools in any session. Ask Claude: "What MCP tools do you have available?" — it will list every tool from every connected server.

💡
Scope Matters: User vs Project
claude mcp add stores the server in your user-level config (~/.claude/settings.json) by default — available in all projects. Use --scope project to add it only for the current project. Use --scope local for local-only (not committed to git).

Lesson 2-2: Team Config with .mcp.json

20 min · Hands-on

User-level MCP config is personal — it doesn't travel with the repo. For team-wide config, use .mcp.json at your project root. When a teammate clones the repo, they get the same MCP servers automatically.

The .mcp.json Format

json 
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "${GITHUB_TOKEN}"
      }
    },
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres", "${DATABASE_URL}"]
    },
    "sentry": {
      "type": "http",
      "url": "https://mcp.sentry.dev/mcp"
    }
  }
}

The ${ENV_VAR} syntax interpolates from environment variables — secrets stay out of the committed file.

Secrets Strategy

Never commit actual tokens to .mcp.json. Use one of these patterns:

bash 
# Option 1: .env file (add to .gitignore)
echo "GITHUB_TOKEN=ghp_..." >> .env

# Option 2: Shell profile (available in all projects)
echo 'export GITHUB_TOKEN=ghp_...' >> ~/.zshrc

# Option 3: Secret manager (best for teams)
# Load from 1Password, AWS Secrets Manager, etc. in your shell profile

What the /from-ticket Skill Now Unlocks

In the Claude for Software Engineers course (module 5), the /from-ticket skill was illustrative — it described fetching ticket data but depended on a human or public URL access. With MCP connected, it becomes real:

bash 
# This now works headlessly — no human needed to paste the ticket
claude -p "/from-ticket LIN-4521" --dangerously-skip-permissions

Claude calls the Linear MCP tool, reads the ticket, reads acceptance criteria, and runs the full /ship pipeline — all unattended.

✅
Checkpoint
Add at least one MCP server (GitHub or Sentry are the easiest to start with), run claude mcp list to confirm it's registered, then ask Claude in a session what tools that server provides. Create a .mcp.json for your project with the servers your team needs.

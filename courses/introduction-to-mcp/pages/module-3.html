Module 3

Building Your First MCP Server

Scaffold a Node.js MCP server from scratch, expose a Tool and a Resource, connect it to Claude Code, and test it end-to-end in under an hour.

2 lessons · ~60 minutes · Hands-on

Lesson 3-1: Scaffold and Expose a Tool

35 min · Hands-on

We'll build a server that searches an internal wiki. By the end of this lesson it will be connected to Claude Code and answering real queries.

1. Project Setup

bash 
mkdir my-mcp-server && cd my-mcp-server
npm init -y
npm install @modelcontextprotocol/sdk zod
npm install -D typescript @types/node tsx

Add a start script and set the module type in package.json:

json 
{
  "type": "module",
  "scripts": {
    "start": "node --loader ts-node/esm src/index.ts",
    "dev": "tsx src/index.ts"
  }
}

2. The Minimal Server

Create src/index.ts:

typescript 
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { z } from "zod";

const server = new McpServer({
  name: "my-mcp-server",
  version: "1.0.0",
});

// ── Tool: search the wiki ────────────────────────────────────
server.tool(
  "search-wiki",
  "Search the internal wiki for documentation on a topic.",
  { query: z.string().describe("Search query") },
  async ({ query }) => {
    // Replace with your real wiki API call
    const results = await searchWiki(query);
    return {
      content: [{ type: "text", text: JSON.stringify(results, null, 2) }],
    };
  }
);

// ── Stub: replace with real wiki search ─────────────────────
async function searchWiki(query: string) {
  return [
    { title: "API Authentication", url: "/wiki/auth", snippet: "..." },
    { title: "Deployment Guide", url: "/wiki/deploy", snippet: "..." },
  ];
}

// ── Start server ─────────────────────────────────────────────
const transport = new StdioServerTransport();
await server.connect(transport);

3. Connect to Claude Code

bash 
claude mcp add my-wiki -- npx tsx /absolute/path/to/my-mcp-server/src/index.ts

4. Test It

Start a Claude Code session and ask:

text 
What does our wiki say about authentication?

Claude will call your search-wiki tool and return the results. If it doesn't, run claude mcp get my-wiki to verify the server registered correctly.

⚠️
Use Absolute Paths
Claude Code spawns the MCP server subprocess from a different working directory. Always use absolute paths in claude mcp add commands, not relative ones. Use $(pwd)/src/index.ts to get the absolute path from your project directory.

Lesson 3-2: Adding a Resource and Multiple Tools

25 min · Hands-on

Now let's extend the server with a Resource (for directly addressable data) and a second Tool, so Claude can both search and fetch specific wiki pages.

Adding a Resource

Resources expose data by URI — like files, but from your system:

typescript 
import { ResourceTemplate } from "@modelcontextprotocol/sdk/server/mcp.js";

// ── Resource: fetch a wiki page by slug ──────────────────────
server.resource(
  "wiki-page",
  new ResourceTemplate("wiki://{slug}", { list: undefined }),
  async (uri, { slug }) => {
    const page = await fetchWikiPage(slug as string);
    return {
      contents: [{
        uri: uri.href,
        mimeType: "text/markdown",
        text: page.content,
      }],
    };
  }
);

async function fetchWikiPage(slug: string) {
  // Replace with your real wiki fetch
  return { content: `# ${slug}\n\nPage content here...` };
}

Adding a Second Tool

typescript 
// ── Tool: list all wiki pages in a section ──────────────────
server.tool(
  "list-wiki-pages",
  "List all wiki pages in a section or category.",
  { section: z.string().describe("Section name, e.g. 'engineering', 'onboarding'") },
  async ({ section }) => {
    const pages = await listWikiPages(section);
    return {
      content: [{ type: "text", text: JSON.stringify(pages, null, 2) }],
    };
  }
);

async function listWikiPages(section: string) {
  return [
    { slug: `${section}/overview`, title: "Overview" },
    { slug: `${section}/setup`, title: "Setup Guide" },
  ];
}

The Full Server at This Point

Your server now exposes:

  • Tool: search-wiki — searches across all pages
  • Tool: list-wiki-pages — lists pages by section
  • Resource: wiki://{slug} — fetches a specific page by slug

Claude can now navigate your entire internal wiki — searching for topics, listing sections, and reading specific pages — all with structured data, no HTML scraping.

💡
Tool Descriptions Drive Auto-Invocation
The description string you pass to server.tool() is how Claude decides when to use the tool. Write it the same way you'd write a skill description — include the use cases and trigger conditions. A good description means Claude reaches for the right tool without being asked explicitly.
✅
Checkpoint
Your server should have at least two tools and one resource. Reconnect it to Claude Code (claude mcp add), then test each tool explicitly: ask Claude to search, to list a section, and to read a specific page by slug. Verify all three return structured data.

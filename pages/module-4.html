Module 4

Hooks — Deterministic Guardrails

Enforce hard constraints on Claude's actions with real code that cannot be hallucinated or ignored — deterministic guardrails for your AI workflow.

2 lessons · ~55 minutes · Intermediate

Lesson 4-1: Hook Types and Lifecycle

25 min · Concept

CLAUDE.md instructions are advisory — they achieve roughly ~80% compliance. Hooks are fundamentally different: they are deterministic. They run real code that cannot be hallucinated, skipped, or misinterpreted.

Lifecycle Events

Hooks fire at specific points in Claude's execution lifecycle:

text 
SessionStart (once)
UserPromptSubmit → [agentic loop] → Stop / StopFailure

Inside each agentic loop turn:
  PreToolUse → [tool executes] → PostToolUse
  PermissionRequest fires on-demand (not a fixed stage)

Five Hook Types

  1. Command hooks — Shell script execution. Exit code 0 = success, exit code 2 = block the action.
  2. HTTP hooks — POST to a URL (CI systems, dashboards, Slack notifications).
  3. MCP Tool hooks — Call an MCP tool from a connected server as the hook action.
  4. Prompt hooks — Single-turn LLM evaluation for fuzzy checks.
  5. Agent hooks — Spawns a subagent with tool access for complex validation logic.

Hook Configuration

Hooks are configured in .claude/settings.json (project-level, commit to git) or ~/.claude/settings.json (user-level, applies to all projects), under the "hooks" key:

json 
{
  "hooks": {
    "PreToolUse": [
      {
        "type": "command",
        "command": "node .claude/hooks/block-dangerous.js",
        "description": "Block force-push and production DB access"
      }
    ]
  }
}

Block Dangerous Commands Script

This hook intercepts tool calls and blocks dangerous commands before they execute:

javascript 
// .claude/hooks/block-dangerous.js
const input = JSON.parse(require('fs').readFileSync('/dev/stdin', 'utf8'));
const cmd = input?.tool_input?.command || '';

const blocked = [
  /git push.*--force/,
  /git push.*-f/,
  /rm -rf \//,
  /DROP TABLE/i,
  /DROP DATABASE/i,
  /psql.*production/,
];

if (blocked.some(pattern => pattern.test(cmd))) {
  console.error('🚫 BLOCKED: Dangerous command detected');
  process.exit(2); // Exit code 2 = block the action
}
process.exit(0); // Allow
🛑
Exit Code 2 = Hard Block
Exit code 2 blocks the action entirely. Claude never runs the command. This is a deterministic guarantee — unlike CLAUDE.md instructions, hooks cannot be ignored or worked around by the AI.

Lesson 4-2: Auto-Lint, Auto-Test, and Audit Hooks

30 min · Hands-on

Now let's build practical hooks that automate quality enforcement: auto-formatting after edits, running tests before session end, and maintaining a complete audit trail.

1. PostToolUse Auto-Lint Hook

Automatically run ESLint with auto-fix after every file edit. Add to .claude/settings.json:

json 
{
  "hooks": {
    "PostToolUse": [
      {
        "type": "command",
        "command": "node .claude/hooks/auto-lint.js",
        "description": "Auto-lint files after edit"
      }
    ]
  }
}
javascript 
// .claude/hooks/auto-lint.js
const { execSync } = require('child_process');
const input = JSON.parse(require('fs').readFileSync('/dev/stdin', 'utf8'));

const filePath = input?.tool_input?.file_path || input?.tool_result?.file_path;
if (!filePath) process.exit(0);

const lintable = /\.(js|ts|jsx|tsx)$/.test(filePath);
if (!lintable) process.exit(0);

try {
  execSync(`npx eslint --fix "${filePath}"`, { stdio: 'pipe' });
  console.log(`✅ Auto-linted: ${filePath}`);
} catch (e) {
  console.warn(`⚠️ Lint issues in: ${filePath}`);
}
process.exit(0);

2. Stop Hook — Run Tests Before Session Ends

Ensure tests pass before Claude completes its work. Add to .claude/settings.json:

json 
{
  "hooks": {
    "Stop": [
      {
        "type": "command",
        "command": "npm test",
        "description": "Run test suite before session ends"
      }
    ]
  }
}

3. PreToolUse Audit Log Hook

Log every tool call to a JSONL file for complete accountability. Add to .claude/settings.json:

json 
{
  "hooks": {
    "PreToolUse": [
      {
        "type": "command",
        "command": "node .claude/hooks/audit-log.js",
        "description": "Log all tool calls to audit trail"
      }
    ]
  }
}
javascript 
// .claude/hooks/audit-log.js
const fs = require('fs');
const input = JSON.parse(fs.readFileSync('/dev/stdin', 'utf8'));

const entry = {
  timestamp: new Date().toISOString(),
  tool: input.tool_name,
  input: input.tool_input,
  session: input.session_id,
};

fs.appendFileSync('.claude/audit.jsonl', JSON.stringify(entry) + '\n');
process.exit(0);
💡
Audit on PreToolUse
Register the audit log on PreToolUse for complete accountability in team environments. This captures every tool call attempt — including ones that get blocked by other hooks — giving you a full picture of what Claude tried to do.
🎯
Checkpoint
Test the auto-lint hook by asking Claude to create a TypeScript file with intentional formatting issues. Verify that ESLint automatically fixes them after the file is written.

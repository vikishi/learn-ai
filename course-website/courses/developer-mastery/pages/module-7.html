Lesson 9-1: Headless Mode & Scheduled Automation

25 min · Hands-on

Headless mode (--print / -p) runs Claude Code non-interactively — perfect for cron jobs, CI pipelines, and automated workflows that don't need human input.

1. Daily Code Quality Report

Schedule a daily quality report that runs every weekday morning:

bash 
#!/bin/bash
# cron: 0 9 * * 1-5 (9 AM weekdays)
cd /path/to/project

claude -p "Generate a daily code quality report:
1. Run the test suite and report coverage
2. Check for TODO/FIXME/HACK comments added this week
3. Identify the 5 most complex functions
4. List any dependency security advisories
5. Save report to reports/daily/$(date +%Y-%m-%d).md" \
  --dangerously-skip-permissions

2. Weekly Dependency Updates

Automate dependency updates with branch creation, testing, and PR submission:

bash 
#!/bin/bash
# cron: 0 10 * * 1 (Monday 10 AM)
cd /path/to/project

BRANCH="chore/deps-update-$(date +%Y%m%d)"
git checkout -b "$BRANCH"

claude -p "Update all dependencies:
1. Run npm outdated and identify safe updates
2. Update patch and minor versions
3. Run the full test suite
4. If tests pass, commit changes
5. Create a PR with a summary of what was updated
6. If tests fail, revert and report which packages broke" \
  --dangerously-skip-permissions

3. Git Worktrees for Parallel Work

Use git worktrees to run multiple Claude instances on different features simultaneously:

bash 
# Create a worktree for a feature branch
git worktree add ../myapp-feature feature-branch
cd ../myapp-feature

# Run Claude non-interactively in the worktree
claude -p "implement the user avatar feature based on ticket #123"

# Meanwhile, continue working in main worktree
# Each worktree gets its own independent Claude instance

Additional Access Methods

  • Mobile access — Connect to Claude Code sessions from your phone for monitoring
  • /teleport — Transfer context between Claude Code sessions seamlessly
  • /remote-control — Allow another developer to guide your Claude instance
⚠️
Use --dangerously-skip-permissions Carefully
The --dangerously-skip-permissions flag bypasses all permission checks. Only use it in trusted, isolated environments (CI runners, dedicated automation servers). Always test your automated workflows interactively first before enabling headless mode.

Lesson 9-2: Team Scaling: Shared Skills, Agents & Plugins

25 min · Hands-on

Your Claude Code configuration becomes infrastructure when shared across a team. Here's how to organize it for maximum reuse and discoverability.

Recommended .claude/ Directory Structure

text 
.claude/
  CLAUDE.md
  settings.json
  agents/
    code-reviewer.md
    security-auditor.md
    test-writer.md
    build-validator.md
    codebase-architect.md
  skills/
    ship/SKILL.md
    code-review/SKILL.md
    generate-tests/SKILL.md
    commit-message/SKILL.md
    security-gate/SKILL.md
    changelog/SKILL.md
    doc-api/SKILL.md
    feature-workflow/SKILL.md
  hooks/
    block-dangerous.js
    auto-lint.js
    audit-log.js

New Developer Onboarding Flow

When this structure is committed to your repository, onboarding becomes trivial:

  1. Clone the repo — All configuration comes with it
  2. Install Claude Code — Extension or CLI
  3. Get all skills/agents/hooks automatically — No manual setup needed
  4. Run /onboard — Generate personalized onboarding docs
  5. Productive on day one — Full automation suite from first commit

Personal vs Project Skills

Location Scope Use Case
.claude/skills/ Project (shared) Team conventions, project-specific automation
~/.claude/skills/ Personal (global) Your preferences, cross-project utilities

Plugins for Cross-Project Skills

Plugins let you share skills across multiple projects without duplicating files. Package common patterns (like your team's PR template, commit conventions, or deployment checklist) as reusable plugins that any project can reference.

💡
Interactive Agent Creation
Use the /agents command for interactive creation — Claude writes sharper descriptions than most developers. The better the description, the more reliably the agent gets invoked.
🎯
Checkpoint
Organize your .claude/ directory following the structure above and commit it to your repository. Verify that a fresh clone picks up all skills and agents.

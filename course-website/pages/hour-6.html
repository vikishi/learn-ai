Part 1: Measuring Effectiveness (20 min)

Create a Review Log

Track your subagent results weekly in review-log.md:

markdown
# Subagent Review Log

## Week 1

### Bugs Caught
1. SQL injection in auth.py — CRITICAL (security-scanner)
   Would have reached production: YES — Time saved: 4+ hours
2. Off-by-one error in pagination — HIGH (code-reviewer)
   Would have reached production: Likely — Time saved: 2 hours

### False Positives
1. Security scanner flagged test code — LOW impact
2. Refactoring assistant too aggressive on 5-line function

### Metrics
- Reviews run: 23
- Average review time: 15 seconds
- Bugs found: 8
- Estimated time saved: 12+ hours
- ROI: 120x (6 min invested → 12 hours saved)

Part 2: Refining Instructions (20 min)

Common Improvements

ProblemFix
Too verbose Add: "Be concise. Only report issues that matter. Skip minor observations."
False positives Add: "Before reporting, verify it's actually wrong and has real impact."
Missing real issues Add specific checks: "ALWAYS check for: null refs, array bounds, error handling"
Inconsistent output Add: "Use EXACTLY this format: [SEVERITY] Line X: [Type]..."

Iteration Process

  1. Use for 1 week — Collect data on performance
  2. Document problems — What's consistently wrong?
  3. Update instructions — Add/modify rules
  4. Test on previous code — Does it work better?
  5. Deploy updated version — Start next iteration

Sharing Options

  • Git commit — Commit .claude/agents/ to your repo. Team members get agents automatically.
  • Personal repo — Keep a separate repo of your agent collection. Copy into new projects.
  • Team template — Create a shared template repo that new projects clone from.
💡
Share via Git

The easiest way to share: commit .claude/agents/ to your project repo. Every team member with Claude Code will automatically see the agents.

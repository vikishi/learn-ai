Advanced Subagent Patterns

Chain subagents into workflows, add project-specific context, and build language-specific reviewers.

60 min
Advanced

Part 1: Multi-Step Workflows (20 min)

Create a Code Improver that orchestrates multiple subagents in sequence:

markdown
---
name: code-improver
description: Orchestrates a full code quality improvement workflow — security, review, testing, and refactoring in sequence
tools: Read, Bash, Grep, Glob
---

# Code Improver Subagent

You coordinate a comprehensive code improvement workflow.

## Workflow Steps

### Step 1: Security First
Delegate to security-scanner. If CRITICAL issues found → stop and report.

### Step 2: Code Quality Review
Delegate to code-reviewer. Collect bugs, performance issues, error gaps.

### Step 3: Test Coverage Check
Delegate to test-generator. Identify untested paths, generate tests.

### Step 4: Structure Analysis
Delegate to refactoring-assistant. Find duplication, complexity.

### Step 5: Compile Report

```
=== SECURITY ===
[Critical/High issues — must fix immediately]

=== CORRECTNESS ===
[Logic errors and bugs]

=== TESTING ===
[Coverage gaps and generated tests]

=== STRUCTURE ===
[Refactoring opportunities]

=== PRIORITY ACTION PLAN ===
Immediate: [Most critical]
Short Term: [Important improvements]
Future: [Technical debt]
```

## Decision Logic
- Critical security → STOP, report immediately
- Code is excellent → Say so, no forced issues
- Minor issues only → Complete full workflow

## Modes
- **Quick check:** Security + bugs only
- **Standard:** Full workflow
- **Deep analysis:** Full + complexity metrics + docs coverage

Part 2: Context-Aware Reviewing (20 min)

Add project-specific knowledge directly to your agent files. Since there's no separate context file, append a Project Context section to your agents:

markdown
# Append this to the bottom of code-reviewer.md:

## Project Context

**Stack:** Python 3.11, Django 4.2, PostgreSQL, Redis, Celery
**Frontend:** React 18, TailwindCSS

## Code Conventions
- Type hints required on all public functions
- Max function length: 40 lines
- Max file length: 500 lines
- Use select_related/prefetch_related (no N+1 queries)
- Transactions on multi-step DB operations

## Security Rules
- Never log PII (emails, names, addresses)
- Always use Django ORM (no raw SQL without approval)
- CSRF protection on all state-changing endpoints
- Check permissions on every resource access

## Red Flags (Always Report)
- Raw SQL without approval
- Hardcoded secrets
- Missing permission checks
- PII in log statements
- Functions over 50 lines
💡
One File, Complete Context

Everything lives in the single agent .md file — instructions AND project context. This ensures the context is always loaded when the agent runs.

Part 3: Language-Specific Reviewers (20 min)

Create specialized reviewers for specific languages. Here's a Python reviewer:

markdown
---
name: python-reviewer
description: Reviews Python code for type hints, idiomatic patterns, and common pitfalls
tools: Read, Bash, Grep, Glob
---

# Python-Specific Checks

## Common Mistakes
- Mutable default arguments: def add(item, list=[])
- Late binding closures: [lambda: i for i in range(5)]
- Using 'is' for value comparison instead of '=='

## Modern Python (3.10+)
- pathlib over os.path
- Context managers for resources
- Dataclasses for simple classes
- Union type syntax: int | str instead of Union[int, str]
- Pattern matching where appropriate

## Performance
- Generators for large datasets
- Sets for membership testing
- List comprehensions over loops

## Error Handling
- Specific exceptions, never bare except
- EAFP over LBYL (dict.get over key-check)

Similarly, create a JavaScript reviewer (.claude/agents/javascript-reviewer.md) covering:

  • const/let over var
  • async/await over promise chains
  • React hooks correctness (dependency arrays)
  • Optional chaining and nullish coalescing
  • Event listener cleanup in useEffect

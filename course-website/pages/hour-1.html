Hands-on

Part 1: What Are Subagents? (15 min)

Subagents are specialized AI assistants that Claude can delegate tasks to. Think of them as expert consultants that Claude calls upon when needed.

Without vs. With Subagents

text
Without: "Review for security, bugs, refactoring, and tests"
→ Claude tries everything at once, unfocused response

With: "Review my authentication code"
→ Security Scanner: Finds 2 vulnerabilities
→ Code Reviewer: Identifies 1 logic bug
→ Test Generator: Creates 8 test cases
→ Claude: "Here's what the specialists found..."

Key Benefits

  • Focused Expertise — Each subagent specializes in one thing
  • Cleaner Conversations — Main chat stays organized
  • Better Results — Specialists dig deeper than generalists
  • Reusable — Write once, use across all projects
  • Customizable — Adapt to your workflow and standards

Part 2: Subagent Architecture (15 min)

Every subagent is a single Markdown file in .claude/agents/:

markdown
---
name: code-reviewer
description: Reviews code for bugs, security issues, and quality
tools: Read, Bash, Grep, Glob
---

# Code Reviewer

You are an experienced software engineer reviewing code...

Required Frontmatter Fields

  • name — Used to identify and invoke the agent
  • description — Claude reads this to decide when to delegate automatically; be specific
  • tools — Comma-separated: Read, Bash, Edit, Write, Grep, Glob

How Claude Activates Subagents

text
# Explicit delegation
You: "Use the security scanner on auth.py"

# Implicit delegation (Claude recognizes the task)
You: "Check this code for security issues"

# Manual control
You: "Review this with the code-reviewer subagent"

Part 3: Your First Subagent (30 min)

Let's build a Bug Finder subagent from scratch.

Step 1: Create Directory Structure

bash
cd ~/your-project
mkdir -p .claude/agents
touch .claude/agents/bug-finder.md

Step 2: Write Instructions

Add this to .claude/agents/bug-finder.md:

markdown
---
name: bug-finder
description: Finds real bugs that cause crashes, wrong results, or unexpected behavior
tools: Read, Bash, Grep, Glob
---

# Bug Finder Subagent

You are a bug-finding specialist focused on identifying real errors in code.

## Your Mission
Find actual bugs that would cause incorrect behavior, crashes, or unexpected 
results. Ignore style issues and theoretical problems.

## Bug Types to Look For

**Critical Bugs:**
- Off-by-one errors in loops/arrays
- Null/undefined/None reference errors
- Division by zero
- Resource leaks (files, connections, memory)
- Race conditions in concurrent code

**Common Bugs:**
- Missing error handling
- Incorrect comparison operators (= vs ==)
- Type mismatches
- Missing return statements

**Logic Bugs:**
- Wrong conditional logic
- Incorrect mathematical operations
- State management errors
- Incorrect variable scoping

## Output Format

For each bug found:
```
[SEVERITY] Line X: [Bug Type]

Problem: [Explain what's wrong]
Impact: [What happens when this runs]
Fix: [Show corrected code]
```

Severity levels:
- CRITICAL - Will crash or corrupt data
- HIGH - Will produce wrong results
- MEDIUM - May fail in edge cases
- LOW - Minor issue with low impact

## What NOT to Report
- Style preferences (naming, formatting)
- Performance optimizations (unless severe)
- Missing documentation
- Theoretical security issues without clear exploit

Step 3: Create Test Code

Create a file with intentional bugs for testing:

python
# buggy_sample.py — Contains 5 intentional bugs

def process_users(users):
    results = []
    # BUG 1: Off-by-one error
    for i in range(len(users) + 1):
        user = users[i]
        # BUG 2: No null check
        email_parts = user.email.split('@')
        domain = email_parts[1]
        results.append(domain)
    return results

def calculate_average(numbers):
    # BUG 3: Division by zero when list is empty
    total = sum(numbers)
    return total / len(numbers)

def find_user_by_id(user_id, users):
    for user in users:
        # BUG 4: 'is' checks identity, not equality
        if user.id is user_id:
            return user
    return None

def save_to_file(data, filename):
    # BUG 5: Resource leak - file not closed
    file = open(filename, 'w')
    file.write(data)

Step 4: Test Your Subagent

text
You: "Use the bug-finder on buggy_sample.py"
